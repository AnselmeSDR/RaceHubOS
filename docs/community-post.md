## Version longue (post de présentation)

🏁 **RaceHubOS — Le gestionnaire de course open source pour Carrera Digital 132/124**

Salut à tous,

Comme beaucoup d'entre vous, on utilise des circuits Carrera Digital 132 depuis des années. Et comme beaucoup, on a vu l'app officielle Carrera disparaître du Play Store, ne plus être mise à jour, et devenir inutilisable sur les appareils récents.

SmartRace fait un bon boulot, mais c'est une app payante, propriétaire et fermée. Pas de personnalisation possible, pas de contrôle sur vos données, et si le développeur arrête un jour... on se retrouve au même point.

On a donc lancé **RaceHubOS** — un système de gestion de course **100% gratuit et open source** qui tourne directement dans le navigateur.

**Pourquoi c'est différent :**
- 🌐 **Web-based** — fonctionne sur n'importe quel appareil avec un navigateur (PC, tablette, téléphone)
- 🆓 **Gratuit pour toujours** — pas de store, pas d'abonnement, pas de version "premium"
- 🔓 **Open source** — le code est ouvert, chacun peut l'adapter à ses besoins
- 🏠 **Vos données restent chez vous** — tout est stocké en local, pas de cloud

**Ce qui marche déjà :**
- Classement en temps réel avec animations de positions
- Gestion pilotes, voitures, circuits et équipes avec photos
- Championnats complets (essais libres, qualifications, course)
- Classements et statistiques par championnat
- Podium en fin de session
- Annonces vocales (meilleur tour, résultats podium)
- Feux de départ avec séquence sonore
- Mode sombre
- Connexion Bluetooth à la Control Unit via AppConnect
- Simulateur intégré pour tester sans matériel

**Ce qu'on voudrait faire avec votre aide :**
- Faire tourner l'app sur un Raspberry Pi pour avoir une station de course dédiée
- Interface tablette optimisée pour la direction de course
- Écrans externes dédiés pour les spectateurs (classement live, timing)
- Builds packagés pour une installation en un clic
- Pénalités, stratégie carburant, relais par équipe, graphiques de tours...

Le projet est jeune mais actif. On est deux pour l'instant — un dev et un passionné de slot qui définit les besoins. On aimerait que la communauté s'en empare.

**Vous pouvez :**
- Tester et nous faire des retours
- Proposer des idées via les issues GitHub
- Contribuer au code si vous êtes dev
- Partager autour de vous

👉 GitHub : https://github.com/AnselmeSDR/RaceHubOS
👉 Contact : anselme8@icloud.com

Merci et bonnes courses ! 🏎️
